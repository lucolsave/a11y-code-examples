    Instructions

    Search Input

    The search form should be wrapped within the <search> element or use the role="search" attribute. The <input> element should use type="search". In addition to this, a couple of extra attributes should be considered:

    • role="combobox": Combines an input field with a popup providing possible values for that input field.
    • aria-activedescendant: An ID pointing to the currently active suggestion (if the user has an active selection).
    • aria-autocomplete="list": Indicates that inputting text could trigger predictions or suggestions. If the suggestion also appears after the caret in the input, use aria-autocomplete="both".
    • aria-expanded: Set to true or false depending on whether the suggestion popover is visible.
    • aria-controls: An ID pointing to the listbox of search suggestions. (Note: This element should always exist in the DOM, even if it is hidden initially).
    • autocomplete="off": Turns off automated assistance in filling out form field values.
    • aria-describedby: Points to an element with visually hidden instructions for screen reader users.

    Suggestions

    Suggestions can be displayed using <ul> and <li> elements, with the following attributes:

    The <ul> element

    • role="listbox": Indicates a list of selectable options.

    The <li> element

    • role="option": Announces the list item as an option to screen readers.
    • aria-selected="true": Add this if the suggestion is automatically added to the input as the user navigates through the list.

    Navigation

    • Enter the proposals: Down arrow (from input to first suggestion)
    • Navigate between proposals: Up and down arrows (no Tab)
    • Exit proposals: Up arrow (from first item back to input)
    • Select a proposal: Use Space or Enter
    • After selection: Automatically reposition focus back in the input
    • Esc key: Closes the listbox and repositions focus in the input
    • No item selected: Focus returns to input without clearing value
    • Left/Right arrows: Move the caret within the input without closing the listbox
    • Tab from combobox: Continues to next focusable element
    • Tab from listbox: Continues focus course, input is filled with current suggestion
    • Shift + Tab from listbox: Moves focus backward, input is filled with current suggestion
